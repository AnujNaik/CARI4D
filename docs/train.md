# Preparing Training Data for CARI4D

## Summary

Training CARI4D requires several categories of data files:

1. **Configuration & split files** that define training hyperparameters and which sequences to use.
2. **Per-sequence data** (rendered H5, packed GT parameters, NLF predictions) for each training/test sequence.
3. **SMPL body models and landmark regressors** for human pose representation.
4. **Object template meshes** from the BEHAVE dataset for 3D object shape.
5. **Auxiliary metadata** (excluded frames/views, object symmetries) for data filtering and loss computation.

An example training command:

```bash
accelerate launch --num_processes=1 learning/training/trainer.py \
    config=learning/configs/cari4d-release.yml \
    split_file=splits/demo-behave.json \
    exp_name=cari4d-train
```

## File-by-File Explanation

### 1. `learning/configs/cari4d-release.yml`

The main training configuration file. It defines:

- **Model architecture**: encoder type, attention layers, feature dimensions, dropout, prediction heads.
- **Training hyperparameters**: learning rate, batch size, number of epochs, scheduler, gradient clipping.
- **Data settings**: input resolution (`224x224`), clip length, sliding window size, augmentation flags.
- **Loss weights**: rotation loss, translation loss, human joint loss, contact loss, etc.
- **Data paths**: `render_root`, `packed_root`, `nlf_root`, `split_file`, `exclude_frames` (all overridable via CLI).

Command-line arguments (e.g., `split_file=splits/demo-behave.json`) override values in this file.

### 2. `splits/demo-behave.json`

A JSON file that defines the train/test split. It lists sequence names under `"train"` and `"test"` keys.

Example content:
```json
{
  "train": ["Date03_Sub03_chairblack_lift"],
  "test": ["Date03_Sub03_chairblack_lift"]
}
```

Each entry is a sequence name that must have corresponding render H5, packed pkl, and NLF pkl files in the data directories.

### 3. `data/train/render/<seq_name>_render.h5`

An HDF5 file containing pre-rendered data for each sequence. For every frame and camera view, it stores:

- **Rendered RGB images** of the object at perturbed poses (used as input image A).
- **Depth maps** rendered from the perturbed object pose.
- **Object masks** (`mask_o`): separate masks for visible and full object regions.
- **Camera-to-world transforms** (`w2c`): rotation and translation for each Kinect camera.
- **Perturbed 6D object poses**: the initial noisy poses that the model refines.
- **Intrinsic matrices** (`K_roi`): camera intrinsics for the cropped ROI.
- **Mesh diameter**, **rotation normalizer**, **translation normalizer**: used to normalize poses to a canonical scale.
- **Input data** (`_input` keys): mixed RGB+mask images (`rgbmB`) and XYZ maps (`xyzB`) from the real observation (image B).

Keys follow the pattern: `<seq>+<frame>_k<kinect_id>_perturb_<perturbation_id>` for rendered data, and `<seq>+<frame>_k<kinect_id>_input` for observation data.

To generate this file, run:
```bash 
python prep/render_fp_nlf.py --nlf_path=data/train/nlf \
 --fp_root=data/train/fp --data_source behave \
 --video <root>/Date03_Sub03_chairblack_lift.0.color.mp4 \
 -o data/train/test --debug
```
Here the nlf and fp results are generated by `prep/run_nlf_sepK.py` and `prep/fp_filter_2dir.py` respectively. 

Note that here all the scripts processe all multi-view videos instead of just a single view, hence the given video path should also have other views in the same folder, e.g. for BEHAVE:
```bash
# View 0
${DIR}/Date03_Sub03_chairblack_lift.0.color.mp4
${DIR}/Date03_Sub03_chairblack_lift.0.depth-reg.mp4
${DIR}/Date03_Sub03_chairblack_lift.0.time.json
# View 1
${DIR}/Date03_Sub03_chairblack_lift.1.color.mp4
${DIR}/Date03_Sub03_chairblack_lift.1.depth-reg.mp4
${DIR}/Date03_Sub03_chairblack_lift.1.time.json
# View 2
${DIR}/Date03_Sub03_chairblack_lift.2.color.mp4
${DIR}/Date03_Sub03_chairblack_lift.2.depth-reg.mp4
${DIR}/Date03_Sub03_chairblack_lift.2.time.json
# View 3
${DIR}/Date03_Sub03_chairblack_lift.3.color.mp4
${DIR}/Date03_Sub03_chairblack_lift.3.depth-reg.mp4
${DIR}/Date03_Sub03_chairblack_lift.3.time.json
```
where `{DIR}` is the root path to the videos, and `depth-reg.mp` is the estimated UniDepth after aligning to GT depth (using `prep/align_monodmap.py`). 

### 4. `data/train/packed/<seq_name>_GT-packed.pkl`

A pickle file (loaded via `joblib`) containing ground-truth annotations for the full sequence:

- `obj_angles`: object rotation as axis-angle vectors, shape `(T, 3)`.
- `obj_trans`: object translation vectors, shape `(T, 3)`.
- `poses`: SMPL body pose parameters (axis-angle), shape `(T, 156)` for SMPL-H.
- `trans`: SMPL global translation, shape `(T, 3)`.
- `betas`: SMPL shape parameters, shape `(T, 10)`.
- `joints_body`: 25 body keypoint locations, shape `(T, 25, 3)`.
- `joints_smpl`: 52 SMPL joint locations, shape `(T, 52, 3)`.
- `frames`: list of frame identifiers (e.g., `t0003.000`).
- `occ_ratios`: per-frame, per-view occlusion ratios, shape `(T, 4)`.
- `dists_h2o`: human-to-object distances per joint, shape `(T, 52)`, used for contact supervision.
For BEHAVE, the packed GT files can be downloaded from [here (part1)](https://datasets.d2.mpi-inf.mpg.de/cvpr22behave/behave-packed-train-seqs-p1.zip) and [here (part2)](https://datasets.d2.mpi-inf.mpg.de/cvpr22behave/behave-packed-train-seqs-p2.zip). 
The `dists_h2o` can then be computed with this:
```bash
python prep/render_dmap_human.py --video <root>/Date03_Sub03_chairblack_lift.0.color.mp4 --packed_root data/train/packed
```


### 5. `data/train/nlf/<seq_name>_params.pkl`

A pickle file containing predictions from the NLF (Neural Lifting Field) human pose estimator, used as initialization for the human branch:

- `poses`: predicted SMPL pose parameters, shape `(T, K, 156)` where `K` is the number of camera views.
- `transls`: predicted SMPL translations, shape `(T, K, 3)`.
- `betas`: predicted shape parameters, shape `(T, K, 10)`.
- `joints_25` (optional, computed on first run): 25 body keypoints derived from the predicted poses, shape `(T, K, 25, 3)`.
- `gender`: subject gender string (`'male'` or `'female'`).
- `frames`: frame identifiers matching the packed file.
- `kids`: list of Kinect camera IDs.

On the first training run, if `joints_25` is absent, the code computes it via forward SMPL and saves it back to this file for faster subsequent loading.

### 6. `data/train/behave-30fps-exclude.json`

A JSON file listing frames that should be excluded from training (e.g., frames with annotation errors or poor quality). Structure:

```json
{
  "frames": ["Date03_Sub03_chairblack_lift/t0005.000/k0", ...]
}
```

Each entry identifies a specific frame and camera view. Referenced by the `exclude_frames` config key. This is only needed for BEHAVE dataset training. 

### 7. `data/train/exclude-views.json`

A JSON file listing sequence+view combinations to skip due to large translation errors or missing masks. Structure:

```json
{
  "exclude": ["Date03_Sub03_chairblack_lift/k2", ...]
}
```

This is hardcoded in `video_data.py` and always loaded. This is only needed for BEHAVE dataset training. 

### 8. `data/train/behave-symmetries.pkl`

A pickle file mapping object names to their symmetry transforms. Each entry is a list of `4x4` transformation matrices representing equivalent poses under the object's symmetry group. Used during loss computation (`symm_loss`) to handle rotationally symmetric objects (e.g., a stool looks the same after 90-degree rotations).

Structure: `dict[str, np.ndarray]` where values have shape `(N_symm, 4, 4)`.

### 9. `data/smpl/SMPLH_male.pkl` and `data/smpl/SMPLH_female.pkl`

SMPL-H body model files for male and female respectively. 

### 10. `data/assets/body25_regressor.pkl`, `face_regressor.pkl`, `hand_regressor.pkl`

Sparse regressor matrices that map SMPL mesh vertices to specific landmark locations:

- **body25_regressor**: maps to 25 OpenPose-style body keypoints.
- **face_regressor**: maps to facial landmarks.
- **hand_regressor**: maps to hand keypoints.

Used by `BodyLandmarks` (in `lib_smpl/body_landmark.py`) to compute the 25 body keypoints from SMPL vertices when `joints_25` is not pre-computed in the NLF file.

### 11. `<BEHAVE_ROOT>/objects/<obj_name>/<obj_name>_f1000.ply`

Simplified (1000-face) mesh templates for each BEHAVE object category. The code loads templates for all 20 BEHAVE objects via `load_templates_all()` regardless of which sequences are in the split. For your own dataset, rewrite the code to load your own mesh files. 

For each template, it computes:

- **Oriented bounding box** (`bbox`, `to_origin`, `extents`): used for evaluation and visualization.

The 20 object categories are: backpack, basketball, boxlarge, boxlong, boxmedium, boxsmall, boxtiny, chairblack, chairwood, keyboard, monitor, plasticcontainer, stool, suitcase, tablesmall, tablesquare, toolbox, trashbin, yogaball, yogamat.

The path is controlled by `BEHAVE_ROOT` in `behave_data/const.py`.
